### 2. Context Engine
**File:** `src/lib/ai/context-engine.ts`

**What it does:**
- Aggregates all business data into structured context
- Calculates business metrics (revenue, retention, no-shows)
- Analyzes customer behavior and segments
- Provides market context and benchmarks
- Caches context for 5 minutes

**Key Functions:**
```typescript
buildAIContext(businessId)        // Build complete context
buildBusinessContext(businessId)  // Business metrics
buildCustomerContext(businessId)  // Customer analysis
buildMarketContext(businessId)    // Market trends
getContextSummary(context)        // Human-readable summary
getCachedContext(businessId)      // Cached version
```

**What it calculates:**

**Business Metrics:**
- Total revenue
- Monthly revenue
- Average transaction value
- Total customers
- Active customers
- Retention rate
- No-show rate
- Average rating

**Customer Segments:**
- VIP (high spend, frequent visits)
- Regular (normal behavior)
- At-Risk (high churn risk)
- Inactive (no visits in 180+ days)
- New (1 appointment or less)

**Customer Behavior:**
- Total appointments
- Completed vs cancelled vs no-shows
- Total spent & average spend
- Days since last visit
- Preferred time and day
- Lifetime value (LTV)
- Churn risk (0-1)

## 🚀 HOW TO USE THE AI REVENUE AGENT

### 1. Access the AI Dashboard

Navigate to: `/admin/ai`

You'll see:
- Revenue generated by AI
- Actions taken and success rate
- Time saved
- Customers affected
- Recent activity feed
- AI insights and recommendations

### 2. Configure AI Settings

Click the "Settings" tab to configure:

**General Settings:**
- Enable/disable AI
- Autonomy level (assistant/copilot/autopilot)
- Max actions per day
- Max spend per action

**Auto-Approval:**
- Low risk actions (reminders, follow-ups)
- Medium risk actions (rescheduling, promotions)
- High risk actions (pricing changes)

**Capabilities:**
- Customer re-engagement
- Intelligent scheduling
- Dynamic pricing
- Upselling
- Customer service
- Marketing campaigns
- Analytics
- Predictions

**Goals:**
- Primary goal (revenue/retention/satisfaction/efficiency)
- Target revenue increase (%)
- Target retention rate (%)

### 3. Run Manual Analysis

Click "Run Analysis" button to:
1. Analyze your business data
2. Generate AI decisions
3. Create actionable insights
4. Queue actions for execution

### 4. Review and Approve Actions

Actions requiring approval will appear in the activity feed with "Pending" status.

### 5. Automated Daily Cycle

The AI runs automatically:
- **2:00 AM** - Analysis runs for all businesses
- **3:00 AM** - Approved actions execute

### 6. Configure API Keys

Add to your `.env`:

```bash
# OpenAI (Primary)
OPENAI_API_KEY=sk-...

# Anthropic (Fallback)
ANTHROPIC_API_KEY=sk-ant-...

# Cron Secret (for automated jobs)
CRON_SECRET=your-secret-key
```

Without API keys, the system uses mock responses for development.

### 7. Set Up Cron Jobs

Configure Cloudflare Workers cron triggers:

```toml
# wrangler.toml
[triggers]
crons = [
  "0 2 * * *",  # AI Analysis at 2 AM
  "0 3 * * *"   # Action Execution at 3 AM
]
```

Or use any cron service to call:
- `GET /api/cron/ai-analysis` (with Bearer token)
- `GET /api/cron/ai-actions` (with Bearer token)
